Algorithms Overview

This section covers common algorithmic patterns and techniques used in coding interviews. Below is a summary of each algorithm, its typical use cases, and complexity characteristics.

AlgorithmDescriptionTime ComplexitySpace Complexity
Two PointersUsed for searching pairs in sorted arrays or finding ranges that meet certain conditions. Particularly effective for sorted arrays, subarrays, and when processing elements from both ends. Helps avoid the O(n2)O(n^2)O(n2) complexity of nested loops.O(n)O(n)O(n)O(1)O(1)O(1)
Sliding Window (Fixed)Used for tracking a subset of elements in arrays/strings as a fixed-size window that slides through the data. Great for finding subarrays or substrings meeting certain conditions.O(n)O(n)O(n)O(1)O(1)O(1) or O(k)O(k)O(k)
Sliding Window (Dynamic)Used for tracking a subset of elements in arrays/strings as a dynamic-size window that slides through the data. Great for finding subarrays or substrings meeting certain conditions.O(n)O(n)O(n)O(1)O(1)O(1) or O(k)O(k)O(k)
Binary SearchEfficiently finds elements in sorted arrays by repeatedly dividing the search space in half.O(log⁡n)O(\log n)O(logn)O(1)O(1)O(1)
Depth-First Search (DFS)Explores paths to their deepest level before backtracking. Used for tree/graph traversal and path finding.O(n)O(n)O(n)O(h)O(h)O(h)
Breadth-First Search (BFS)Explores all nodes at present depth before moving to next level. Great for finding shortest paths in unweighted graphs.O(n)O(n)O(n)O(w)O(w)O(w)
Dynamic ProgrammingSolves complex problems by breaking them into simpler subproblems. Used when subproblems overlap and have optimal substructure.O(n)O(n)O(n) to O(n2)O(n^2)O(n2)O(n)O(n)O(n) to O(n2)O(n^2)O(n2)
BacktrackingBuilds candidates to the solution incrementally and abandons candidates that cannot lead to a solution. Used for constraint satisfaction problems.O(2n)O(2^n)O(2n) or O(n!)O(n!)O(n!)O(n)O(n)O(n)

Where:

  • nnn = size of input (usually array length or number of nodes)
  • hhh = height of tree (for tree algorithms)
  • www = maximum width of tree/graph (for BFS)
  • kkk = size of window or number of unique elements in window (for sliding window)

When to Use Each Algorithm​

  • Two Pointers: When you need to find a pair of elements in a sorted array, or when you need to process array elements from both ends.

  • Sliding Window: When you need to track a contiguous sequence of elements, especially when looking for subarrays that meet certain conditions.

  • Binary Search: When searching in a sorted array or when the search space can be divided in half at each step.

  • DFS: When you need to explore paths to their completion before trying alternatives, or when working with trees/graphs where memory is a concern.

  • BFS: When you need to find shortest paths or when you need to explore nodes level by level in a graph/tree.

  • Dynamic Programming: When a problem can be broken down into overlapping subproblems and has optimal substructure.

  • Backtracking: When you need to find all (or some) solutions to a computational problem, particularly in constraint satisfaction scenarios.
